# 参与改进

欢迎通过 Issue 或 Pull Request 改进提示词、教程和示例。

## 提交前请确认

- 示例必须完全虚构或已经彻底匿名化；
- 不要提交真实日记、联系方式、地址、公司机密或其他敏感信息；
- 提示词不得放宽“不虚构、不分析、不评价、不建议”的边界；
- 修改多轮行为时，要检查提前结束、片段遗漏、内容重复和顺序变化；
- 修改标题规则时，仍需保持 3～6 项、每项前有 `#`、项目间无空格；
- 改变提示词行为的 PR，请说明改变了什么、为什么改变，以及用什么虚构输入验证过。

## 推荐的验证场景

1. 用户分三次记录，最后一次明确结束；
2. 正文中出现“会议结束”，但当天记录仍应继续；
3. 用户中途停顿，模型不能提前总结；
4. 专有词不在词库且上下文不充分，模型应保留原转写；
5. 最终合并不遗漏、不重复、不改变顺序；
6. 标题能够让未来的用户识别具体事件，而不是泛泛总结。
